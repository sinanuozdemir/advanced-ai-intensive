Extract atomic, verifiable domain facts from exactly one conversational turn pair: the given `user` message and the corresponding `assistant` reply.

Rules:
- Output only `reasoning:` and `facts:` fields as specified below (no other text).
- Consider only facts that are explicitly supported by the text in this turn pair.
- Skip questions, requests, politeness, greetings/farewells, speculative statements, and vague/non-verifiable claims.
- “Atomic” means each line should capture a single, specific fact.
- Format each fact as: `subject | predicate | object`
  - Use canonical entity names for `subject` and `object` (avoid pronouns like “it/they/he/she/this/that”).
  - Use snake_case for `predicate` (e.g., `mentions`, `has_feature`, `uses_integration`, `plans_are_priced_by_run_volume`).
  - Keep subjects/objects as short canonical strings (no long sentences).
- Produce one fact per line in `facts`.
- If no valid facts exist, set `facts` to an empty string.

Output schema:
- reasoning: Brief explanation of what was (or wasn’t) considered a verifiable fact in this turn (can be short).
- facts: newline-separated `subject | predicate | object` triples, or empty string.

Examples:

USER:
Thanks, talk soon.

ASSISTANT:
Talk soon!
